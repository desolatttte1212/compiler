# Спецификация языка (MiniCompiler)

## 1. Лексическая грамматика (EBNF)

Лексическая структура языка определяется следующими правилами.
Пробельные символы и комментарии игнорируются сканером.

### 1.1 Базовые символы
letter = 'a' | ... | 'z' | 'A' | ... | 'Z';
digit = '0' | ... | '9';
any_char = любой допустимый символ UTF-8;

### 1.2 Пробелы и комментарии
Пробельные символы пропускаются. Комментарии приравниваются к пробелам.

whitespace = ' ' | '\t' | '\r' | '\n';
comment = single_line_comment | multi_line_comment;
single_line_comment = "//" { любой_символ_кроме_новой_строки } '\n';
multi_line_comment = "/*" { любой_символ | '\n' } "*/";

### 1.3 Идентификаторы и ключевые слова
Идентификаторы начинаются с буквы или подчеркивания, за которыми следуют буквы, цифры или подчеркивания.
Ключевые слова являются зарезервированными идентификаторами и не могут использоваться как имена переменных.

identifier = (letter | '_') { letter | digit | '_' };
keyword = 'if' | 'else' | 'while' | 'for' | 'int' | 'float' | 'bool' | 'return' | 'true' | 'false' | 'void' | 'struct' | 'fn';

**Ограничения:**
- Максимальная длина идентификатора: 255 символов.
- Регистрозависимость: `Var` и `var` считаются разными именами.

### 1.4 Литералы

#### Целочисленные литералы
Десятичная последовательность цифр.
Диапазон: [-2^31, 2^31-1] (проверяется семантически, лексер читает положительную последовательность).
Отрицательные числа разбираются как унарный минус, за которым следует целочисленный литерал.

integer_literal = digit { digit };

#### Вещественные литералы
Последовательность цифр, точка и снова цифры. После точки обязательно должны быть цифры.

float_literal = digit { digit } '.' digit { digit };

#### Строковые литералы
Последовательность символов в двойных кавычках. Поддерживаются экранированные последовательности: \", \\, \n, \t.
Переносы строк внутри строк должны быть экранированы.

string_literal = '"' { character | escape_sequence } '"';
escape_sequence = '\' ('"' | '\' | 'n' | 't');
character = любой_символ_кроме_кавычки_и_обратного_слэша;

#### Булевы литералы
Представлены ключевыми словами `true` и `false`.

bool_literal = 'true' | 'false';

### 1.5 Операторы и разделители

#### Арифметические операторы
arithmetic_op = '+' | '-' | '*' | '/' | '%';

#### Операторы сравнения
relational_op = "==" | "!=" | '<' | "<=" | '>' | ">=";

#### Логические операторы
logical_op = "&&" | "||" | '!';

#### Оператор присваивания
assignment_op = '=';

#### Разделители
delimiter = '(' | ')' | '{' | '}' | ';' | ',';

### 1.6 Структура токена
Токен, генерируемый сканером, состоит из:
- **Тип (Type)**: Значение перечисления (например, KW_IF, INT_LITERAL, IDENTIFIER).
- **Лексема (Lexeme)**: Исходная подстрока из кода.
- **Позиция (Position)**: Номер строки (с 1), Номер колонки (с 1).
- **Значение (Value)**: Разобранное семантическое значение (например, число int, float, содержимое строки без кавычек, булево значение).

## 2. Набор символов и кодировка
- **Кодировка**: UTF-8.
- **Концы строк**: Поддерживаются Unix (\n) и Windows (\r\n). Символы возврата каретки нормализуются.

## 3. Обработка ошибок
- **Недопустимые символы**: Вызывают сообщение об ошибке с координатами строки/колонки. Сканер пропускает недопустимый символ и продолжает работу (режим восстановления).
- **Незавершенные строки**: Вызывают ошибку, если отсутствует закрывающая кавычка или встретился неэкранированный перенос строки.
- **Незавершенные комментарии**: Вызывают ошибку, если достигнут конец файла (EOF) до появления `*/`.
- **Некорректные числа**: Вызывают ошибку для недопустимых последовательностей (например, несколько точек).
